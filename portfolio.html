Flipped (Game)

For an iPhone programming class final project my partner Randy Dang and I created a simple platformer game where the player physics depend on the physical orientation of the device.

Our plan was to make a unique 2D platformer game using Swifts sprite kit. Players will navigate from left to right through various levels using an on screen D-pad, jumping on platforms and maneuvering around deadly obstacles in order to reach the level exit and progress the game.

What makes this game different from other platformers is that it takes advantage of the phones physical orientation. Each level the player will need to rotate the screen several times to reach the exit. The characters size and jump height are relative to the screen height, while in the landscape orientation the player will be able to fit under smaller gaps but will have a lower jump height. When the phone is in portrait mode the player can jump higher and has a taller sprite. Levels are designed in such a way to take advantage of this mechanic.

To complete each level the player must jump, run and rotate to win!

This game was made mostly as a proof of concept. Our goal was to get the game mechanics down without much focus on the visuals

File Transfer Program

For a computer networks class my partner Luka Sitas and I created a program that can be used to transfer files accross a network.

The program transfers files using a TCP connection. It has two modes, reliable and unreliable. These modes are used to simulate a good and a bad internet connection to demonstrate that the program can handle packet loss. With the program in the unreliable mode every tenth packet is lost and another has to be sent using TCP protocols.

Simple Image Classifier

For an image processing class my partners Randy Dang, Kevin Liang and I, were tasked with making a simple image classifier capable of identifying pictures of numbers.

There are 100 total images included. 10 of each digit. 5 images for each digit are randomly selected to be used for training data and the remaining 50 images are used for testing the classifier.

The classifier uses a simple algorithm where the image is divided up into 9 sections and a feature vector is created that stores the ratio of black pixels to white pixels for each section. Feature vectors for each digit are created from the training data and are used to classify testing data.

Infrared Light Switch

I had a stick-on desk lamp in the corner of my room that I would use as a reading light. To turn it on I would walk across my room and flip a switch. To simplify this task I needed a way to control this lamp remotely and infrared seemed to be the best method.

The lamp is powered by a USB drawing about 0.4 Amps at 5V. When the Arduino receives any infrared signal (this makes it, so it can be used with any infrared remote) or the button is pressed it flips a relay and powers the USB port.

To keep the power consumption low and the size of the device small, I used a standalone Arduino chip with a 16mHz resonator.

Automatic Room Heater

For my indoor hydroponics system I was running into some issues with the temperature. My hydroponics system is setup in a room in the basement of my house, as a result it can get quite cold down there if no extra heating is used. If the room temperature drops below 18°C plant growth will slow down significantly. From what I understand the ideal temperature range for plant growth is 22°C to 26°C so I built this device to keep the room within that range. The device takes temperature readings from a DHT11 temperature sensor, outputs the reading in celsius to two 7-segment displays and if the temperture is below the lower threshold it will turn on a space heater until the upper threshold is reached.
